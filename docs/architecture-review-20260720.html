1. Name and deepen the runtime roles

Strong ports & adapters

After — protocol roles stay local

Problem

Python assumes a persistent socket and sends an unanswered ping; Ruby closes every client after one request. Protocol ownership and server/client names both leak.

Solution

Concentrate each protocol role in a named deep module and make the private TCP seam explicit.

Deletion test

Deleting the bridge client would spread framing, retries, and lifecycle back across every MCP tool.

  • locality: protocol bugs concentrate
  • leverage: tools share one bridge
  • tests hit one seam
  • README names stay stable

Problem

Every Ruby path reaches SketchUp globals, UI state, or sockets; the existing “tests” only print live results.

Solution

Concentrate command behavior behind the runtime seam, then use a fake adapter headlessly and a production adapter in TestUp.

Coverage gate

Measure Python and Ruby separately at 100%; retain an in-SketchUp integration suite for adapter truth.

  • interface is test surface
  • locality: failures isolate
  • coverage runs headlessly
  • TestUp checks real adapter
